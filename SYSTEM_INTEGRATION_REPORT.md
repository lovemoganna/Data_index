## 🎯 用户使用指南

### 如何切换指标模式

1. **启动应用**: 运行 `npm run dev`
2. **进入设置**: 点击顶部导航栏的"设置"按钮
3. **选择模式**:
   - **基础模式**: 推荐新用户，包含40个核心指标
   - **完整模式**: 高级用户，包含173个专业指标
4. **确认切换**: 点击切换按钮，系统将自动重新加载

### 模式差异说明

| 特性 | 基础模式 | 完整模式 |
|------|---------|----------|
| 指标数量 | 40个 | 173个 |
| 覆盖维度 | 4个 | 8个 |
| 技术复杂度 | 简单 | 高级 |
| 适用场景 | 学习/基础监控 | 专业风险管理 |
| 系统要求 | 标准 | 较高配置 |

### 数据持久化

- 用户的模式选择会自动保存到浏览器本地存储
- 系统重启后会恢复用户的最后选择
- 支持通过环境变量强制设置默认模式

## 🚀 部署建议

### 生产环境部署

```bash
# 构建生产版本
npm run build

# 部署到服务器
# 确保服务器支持现代JavaScript特性
```

### 环境变量配置

```bash
# 强制使用完整模式（生产环境）
REACT_APP_DATA_MODE=full

# 强制使用基础模式（测试环境）
REACT_APP_DATA_MODE=basic

# 不设置则使用用户偏好或默认基础模式
```
